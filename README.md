# ClassCtl — שליטה על מחשבי כיתה

כלי ניהול לכיתות מחשבים: מכל עמדה בכיתה אפשר לכבות / לאתחל / לאפס כרטיס רשת /
להריץ סקריפט על **שאר המחשבים בכיתה** — עם הגנת סיסמה ואישור לפני ביצוע.
מיועד ל‑Windows 11 (ותומך גם בלינוקס).

## איך זה עובד

שני רכיבים:

- **Agent (סוכן)** — רץ כשירות רקע (הרשאות SYSTEM) על כל מחשב. מבצע פקודות מקומית
  ומשיב לגילוי. מכניסים אותו לגלופה (image), וכל מחשב משוכפל מקבל אותו.
- **Controller (שלט)** — התוכנה עם חלון הסיסמה והכפתורים. רצה מכל עמדה, מגלה את
  מחשבי הכיתה ושולחת להם פקודות.

**זיהוי כיתה** נעשה לפי **שם המחשב** — השם ללא מקטע העמדה, כלומר ללא החלק
שאחרי המקף האחרון (`101-12`→`101`, `303-ins`→`303`, `5500-01-INS`→`5500-01`).
כך שתי כיתות של אותו קורס נשארות נפרדות. זה עובד גם כשכמה
כיתות חולקות אותו VLAN: השלט משדר לרשת, אבל מסנן רק מחשבים עם אותה קידומת כמו
המחשב שממנו הוא רץ, ולא כולל את עצמו.

**הכל סקריפט.** אין פעולות "מובנות". כיבוי/ריסטארט/איפוס‑רשת הם קבצי סקריפט
בתיקייה המוגנת. כל קובץ בתיקייה = כפתור בשלט. מוסיפים קובץ → מופיע כפתור; מוחקים
קובץ → הכפתור נעלם. כשמריצים סקריפט, השלט שולח גם את תוכנו, אז התיקייה בכל
המחשבים נשארת מסונכרנת.

## דרישות

- Python 3.8+ (ספריית התקן בלבד — אין `pip install`). או exe עצמאי (ראה בהמשך).
- מחשבי כיתה על אותו סוויץ'/רשת (broadcast מגיע ביניהם).

## אבטחה — קרא לפני התקנה

- כל הפקודות **חתומות ב‑HMAC** במפתח רשת משותף (`net_key`) שנוצר בהתקנה. בלי המפתח
  אי אפשר לשלוח פקודות. יש הגנת חותמת‑זמן ו‑replay.
- ה‑`net_key` נמצא ב‑`agent.json` על כל מחשב. **חובה** שהתיקייה תהיה מוגנת ב‑ACL
  (האשף עושה זאת) כדי שסטודנט לא יקרא את המפתח. בלי זה — סטודנט יכול לאתחל מחשבים.
- השלט קורא קונפיג מהתיקייה המוגנת — **הרץ אותו כמנהל** (הקיצור עושה זאת אוטומטית).
- **מגבלה חשובה:** אם *כל* משתמשי התחנה הם אדמין (כמו בהרבה מעבדות), ה-ACL אינו
  מחסום מוחלט מולם — אדמין יכול לקחת בעלות ולקרוא את `net_key`. הסיסמה עוצרת שימוש
  מקרי ומשתמשים לא-טכניים, אך לא אדמין נחוש. חסימה מלאה תדרוש מפתח פרטי שנמצא רק
  אצל ה-IT (ואז אי אפשר לשלוט "מכל עמדה"). זו פשרה מודעת.
- כשלים מוצגים בשלט מיד אחרי כל פעולה — מי לא ענה, ה-IP שלו והסיבה, עם אפשרות לנסות שוב.
- כדאי שהכיתות יהיו על VLAN מבודד מרשת הסטודנטים הכללית.

## התקנה (על גלופת המאסטר)

**הדרך הקלה — installer אחד:** הפעל את `install.bat` (קליק כפול). הוא מעלה הרשאות
ועושה הכל: בודק Python (ואם חסר — מציע להתקין דרך winget או לעצור עד שתתקין ידנית),
מעתיק קבצים, מריץ את אשף ההגדרה, מתקין את הסוכן + חומת אש, יוצר קיצורי דרך לכל
המשתמשים, רושם ב"הוספה/הסרה של תוכניות", ומריץ בדיקת בריאות.

לאחר מכן: אטום (sysprep) ושכפל את הגלופה. כל המחשבים מקבלים אותו `net_key` ואותה
סיסמה, אז כל עמדה יכולה לשמש כשלט.

**התקנה ידנית (שלב-שלב), אם מעדיפים:**
1. התקן Python 3.
2. שים את הריפו ב-`C:\ProgramData\ClassCtl`.
3. הרץ כמנהל: `python setup_wizard.py` (סיסמה + תיקייה + סקריפטים בסיסיים).
4. הרץ כמנהל: `.\install_agent.ps1` (Scheduled Task כ-SYSTEM + חומת אש).
5. הרץ את השלט כמנהל: `python controller.py`.

## שימוש יומיומי

1. הפעל את השלט (כמנהל) → הזן סיסמה.
2. השלט מציג את מחשבי הכיתה שנמצאו.
3. לחץ על כפתור (כיבוי / ריסטארט / איפוס רשת / כל סקריפט) → **אשר** → מבוצע על כולם.
4. תוצאות (הצליח/נכשל לכל מחשב) מוצגות בסיום.

**הוספת/הסרת פעולה:** לחץ 📂 לפתיחת התיקייה, הוסף/מחק קובץ סקריפט, ואז ⟳ כפתורים.

## פעולות מוכנות

בהתקנה (וגם ב"הוספת פעולות") אפשר לסמן אילו פעולות לייצר:

| פעולה | Windows | לינוקס |
|---|---|---|
| כיבוי / הפעלה מחדש | `shutdown.ps1` / `restart.ps1` | `.sh` |
| איפוס כרטיס רשת | `reset-network.ps1` | `.sh` |
| סגירת חלונות | `close-windows.ps1` | `.sh` |
| הערת מסכים | `wake-screens.ps1` | `.sh` |
| פתיחת Word / Excel / PowerPoint | `open-word.ps1` וכו' | LibreOffice |

פעולות פתיחת התוכנה פותחות **מסמך ריק חדש**, לא את מסך התבניות. ב‑Windows זה
נעשה דרך COM (`Documents.Add()`), כי `Start-Process winword.exe` נוחת על גלריית
התבניות, ול‑Excel אין מתג שורת‑פקודה מקביל. אם Office לא מותקן — נופל אחורה
להפעלה רגילה. בלינוקס `soffice --writer/--calc/--impress` כבר עושים בדיוק את זה.

הן **לא מסומנות כברירת מחדל** — לא בכל חדר מותקן Office.

## הערת מחשבים (Wake-on-LAN)

מחשב כבוי לא עונה לגילוי, אז צריך להכיר את כתובת ה‑MAC שלו מראש. זה קורה מעצמו:

1. כל סוכן מדווח את ה‑MAC של **כרטיס הרשת שנושא את כתובת ה‑LAN** (לא של מתאם
   וירטואלי של Hyper-V/VMware/VirtualBox — מנת קסם לשם לא מעירה כלום).
2. השלט אוסף את הכתובות בכל סריקה ושומר אותן ב‑`stations.json`.
3. **הטבלה נדחפת חזרה לכל הסוכנים**, כך שכל עמדה יכולה להעיר את הכיתה — לא רק זו
   שבמקרה הריצה את הסריקה. הרשומה החדשה גוברת, אז מחשב שהוחלף מעדכן את עצמו.

בשלט: כפתור **הערה** ליד סריקה מחדש. הוא שולח מנת קסם לכל עמדה מוכרת שאינה עונה.

**לפני שזה עובד:** צריך להפעיל WoL ב‑BIOS, ולהריץ פעם אחת על הגלופה את
`extras/prepare-wol.ps1` (מכבה Fast Startup ומאפשר הערה במנת קסם).

## עדכון

```
git pull
```
(אם שינית את הסוכן — הפעל מחדש את המשימה: `Restart-ScheduledTask -TaskName "ClassCtl Agent"`.)

## הסרה

```
.\uninstall.ps1
```
עוצר ומוחק את המשימה, מסיר את חוקי חומת האש, ומוחק את התיקייה המוגנת (כולל שחזור
בעלות כדי לאפשר מחיקה). זו הדרך היחידה למחוק את התיקייה המוגנת.

## לינוקס

```
sudo python3 setup_wizard.py
sudo ./install_agent_linux.sh          # שירות systemd
sudo systemctl status classctl-agent
```

## exe בלי Python (דרך ב')

על מכונת פיתוח: `pip install pyinstaller` ואז `.\build.ps1`. מעלים את ה‑exe
ל‑GitHub Release, ובגלופה: `.\install_agent.ps1 -AgentExe classctl-agent.exe`.

|---|---|---|
